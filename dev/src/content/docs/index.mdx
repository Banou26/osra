## What is osra?

osra is a zero-runtime-dependency TypeScript RPC library. Both sides call `expose(value, { transport })` and each receives the other's value live: functions stay callable, generators stream, errors and aborts propagate, across Workers, iframes, WebSockets, extensions, or any custom channel.

## Both sides call `expose()`

One object crosses the boundary alive: functions stay callable, generators stream, and every type survives the trip. Hover the code: these are the real types, compiled against the published package.

```ts twoslash title="worker.ts + main.ts"
// ---cut-start---
// @filename: worker.ts
// ---cut-end---
// worker.ts
import { expose } from 'osra'

const api = {
  hello: async (name: string) => `hello ${name}`,
  countTo: async function* (n: number) {
    for (let i = 1; i <= n; i++) yield i
  },
}
export type Api = typeof api

expose(api, { transport: globalThis })

// ---cut-start---
// @filename: main.ts
// ---cut-end---
// main.ts
import type { Api } from './worker'
import { expose } from 'osra'

const worker = new Worker('./worker.js', { type: 'module' })
const remote = await expose<Api>({}, { transport: worker })

await remote.hello('osra') // 'hello osra'
for await (const n of await remote.countTo(3)) console.log(n) // 1 2 3
```
